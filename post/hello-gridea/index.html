好了——接下来想聊一下自己搭建博客的经历和想法吧。

说起来自己也想不清楚为什么自己会对搭建博客这件事有执念……因为跟风？或者是真的想在互联网上留下一块自己的空间？也许还有其他什么原因……？

回到上大二之前的那个暑假，还记得暑假快结束那会尝试了下在 GitHub 上面申请学生包来着…… GitHub 给送了一大堆东西，里面就有 DigitalOcean 的 100 美刀代金券和 Namecheap 的一面免费域名。送了这么多东西可不能浪费——这么想着，之后就照着中文互联网上的一大堆教程研究了好几天，算是把博客搭好了。

第一次用的是 Wordpress ，然后因为想用一个花里胡哨的二次元主题又折腾了半天，这才看起来比较像样。不过这一年过去也没在博客上写些什么文章（

直到有一天，Namecheap 发来邮件说域名到期了……这时候自己才想起来还有个博客（呆

当时就有点摆烂的想法了，域名出售就出售吧，不管了；服务器也一样，一百刀花完就完了，就这样吧。结果服务器的那个实例也是去年年底收到邮件说要欠费了自己才去删……

域名到期后又过了几个月，自己因为觉得折腾 Wordpress 、服务器和域名太麻烦，而这时又看到好多人用 Hexo ——文章保存在本地也安全，没有域名也可以用 GitHub Pages 部署什么的。正好那时候学校的事情还比较少，自己一拍脑袋就开始试着用 Hexo 搭博客了。

结果是，因为自己太菜，哪一步做错了也不知道该怎么处理，GitHub 的仓库让自己给删了三四回；Hexo 的官方文档也不想再看了，要了解的东西有点多而且文档有点旧；弄了一个主题，因为当时还没学 CSS ，自己想去摸索但是一直也弄不好……然后博客这事就又不了了之了。

前几天刷推又刷到了 Gridea ，看到有 GUI 又可以非常方便地部署，于是自己又心动了（（（，遂又把之前从 GitHub 里面建的仓库删掉，开始部署了 Gridea。之后的效果就是现在这个样子啦。

之后自己应该不会再去折腾主题和 CSS 了吧，毕竟这个挺浪费时间。既然是博客，那还是要专注在内容上面的。后面放在博客上的时间和精力大概都会以内容为主了。
